# Offer Lens — Project Reference

## Stack

Vanilla HTML/CSS/JS. No framework, no bundler, no transpilation. Files are served directly by Netlify.

## Key Decisions

**API key delivery:** User enters their Anthropic key in a UI field on first visit. Stored in `localStorage` under `ol_api_key` — persists across sessions until explicitly cleared via "Change API key". Never hardcoded, never committed.

**Module system:** `package.json` has `"type": "module"` so ESLint/stylelint configs can use ESM (`export default`). Browser JS files (`js/*.js`) use classic script mode — ESLint is configured with `sourceType: 'script'` for those files. Do NOT add `export`/`import` to browser JS.

**CSS approach:** Logical properties throughout (`margin-inline`, `padding-block`, etc.). Properties alphabetized per stylelint-order. No `!important` except `.hidden` utility (suppressed in VS Code via `.vscode/settings.json`).

## Known Gotchas

- Lucide icons injected via `innerHTML` (flags, negotiation) require a manual `lucide.createIcons()` call after render.
- `role="status"` on the loading skeleton carries implicit `aria-live="polite"` — do not add `aria-live` separately.
- Skip link target (`#main-content`) has `tabindex="-1"` to accept programmatic focus; `output-heading` also has `tabindex="-1"` for post-render focus.
- `deno.lock` is generated by Netlify CLI — it's in `.gitignore`.

## Pre-Commit

Husky + lint-staged runs on commit:

- `stylelint --fix` + `prettier --write` on CSS
- `eslint --fix` + `prettier --write` on JS
- `html-validate` + `prettier --write` on HTML

Run manually: `npm run lint:css`, `npm run lint:js`, `npm run lint:html`
